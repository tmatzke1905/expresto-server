# Release Notes — 1.0.0-beta

Release date: planned after completion of the beta release gate

Status: first public npm prerelease

## Summary

`1.0.0-beta` is the first supported public prerelease of expresto-server.

This release freezes the realistic v1 core that is already implemented,
packaged, tested, and documented. It is intended for early adopters, internal
production validation, and integration feedback before the stable `1.0.0`
release.

## Highlights

- npm packaging verified for CommonJS and ESM consumers
- stable package-root API documented for the v1 core
- fail-closed JWT and Basic Auth behavior
- protected ops endpoints for production deployments
- runtime wiring aligned with the documented scheduler and config contract
- file-based controllers, lifecycle hooks, EventBus, and ServiceRegistry
- Prometheus metrics and OpenTelemetry request tracing
- attached and standalone scheduler runtime
- optional Socket.IO support on the shared HTTP server

## Supported Scope in 1.0.0-beta

The supported prerelease scope is the v1 core documented in:

- [public-api.md](../public-api.md)
- [framework-contracts.md](../framework-contracts.md)
- [configuration.md](../configuration.md)
- [security.md](../security.md)
- [scheduler.md](../scheduler.md)

## Explicitly Out of Scope

The following areas are not part of `1.0.0-beta`:

- stable example project requirement for release promotion
- plugin loading and plugin configuration
- full multi-process cluster runtime
- public Socket.IO server accessor API

These items remain scheduled for later roadmap packages.

## Operational Notes

- minimum supported Node.js version: `22.x`
- `npm test` now runs once and exits
- `npm run test:watch` is the interactive watch mode
- `npm run coverage` enforces at least 85% statements, functions, and lines
- production deployments must disable ops endpoints or protect them with
  `basic` or `jwt`

## Promotion Criteria for 1.0.0

`1.0.0-beta` must not be promoted to stable until:

- the example project is complete
- the example uses only the published package surface
- the example covers the supported feature set of `1.0.0`
- the documentation points to the example as the canonical starting point

See [releases.md](../releases.md) and [roadmap.md](../roadmap.md) for the
promotion path.

## Verification Summary

The release gate for this prerelease includes:

- package smoke tests against the packed npm artifact
- build verification
- full test suite
- coverage verification
- release-scope and contract documentation review
