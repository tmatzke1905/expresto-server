# Release Notes — 1.1.0-beta

Release date: 2026-03-23

Status: current public prerelease

## Summary

`1.1.0-beta` is the current supported public prerelease of expresto-server.

This update builds on `1.0.0-beta.1` and introduces the first supported
clustered runtime preview for local multi-core deployments. It is intended for
integration testing of the new primary/worker bootstrap, operational review,
and release validation before the corresponding stable minor release.

## Highlights

- real clustered primary/worker runtime via the bundled CLI bootstrap
- leader-only attached scheduler execution in clustered deployments
- graceful worker shutdown with restart supervision in the primary process
- worker-local ops and health output with explicit cluster metadata
- additional Prometheus cluster metadata gauges
- clear startup errors for unsupported `cluster + websocket` and
  `cluster + scheduler.mode=standalone` combinations

## Explicitly Out of Scope

The following areas are not part of `1.1.0-beta`:

- stable example project requirement for release promotion
- plugin loading and plugin configuration
- clustered WebSocket deployments
- distributed cross-host cluster coordination

These items remain scheduled for later roadmap packages or future stable
release work.

## Operational Notes

- minimum supported Node.js version: `22.x`
- `cluster.enabled` is activated by the bundled CLI bootstrap, not by
  `createServer()` imports alone
- only one designated cluster worker runs attached scheduler jobs
- ops and Prometheus responses in cluster mode are worker-local, not aggregated
- `npm run coverage` currently verifies about 91% statements, 91% functions,
  and 91% lines

## Promotion Criteria for 1.1.0

`1.1.0-beta` should only be promoted once:

- clustered runtime behavior remains backwards compatible for existing
  single-process apps
- shutdown and worker restart behavior are validated in automation
- the final release docs still match the shipped cluster behavior exactly
- the stable release plan for `1.0.0` and the example-app gate are resolved
  explicitly

See [releases.md](../releases.md) and [roadmap.md](../roadmap.md) for the
version and package mapping.
