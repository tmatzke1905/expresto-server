# Release Notes — 1.0.0-beta.1

Release date: 2026-03-22

Status: published historical follow-up prerelease

## Summary

`1.0.0-beta.1` was the supported public follow-up prerelease of
expresto-server.

This update builds on `1.0.0-beta` and adds the supported WebSocket runtime
extension API that was intentionally held back from the first beta publish. It
is intended for continued integration testing, early adopter feedback, and
final stabilization work ahead of the stable `1.0.0` release.

## Highlights

- supported `ExprestoRuntime.getSocketServer()` access after HTTP startup
- shared Socket.IO server wiring on the first `app.listen(...)`
- documented behavior for runtimes before startup and with WebSockets disabled
- expanded runtime and public API coverage for WebSocket and CLI startup paths
- coverage thresholds enforce at least 85% statements, functions, and lines

## Explicitly Out of Scope

The following areas are not part of `1.0.0-beta.1`:

- stable example project requirement for release promotion
- plugin loading and plugin configuration
- full multi-process cluster runtime

These items remain scheduled for later roadmap packages or the stable release
gate.

## Operational Notes

- minimum supported Node.js version: `22.x`
- `npm test` runs once and exits
- `npm run test:watch` is the interactive watch mode
- `npm run coverage` enforces at least 85% statements, functions, and lines
- `runtime.getSocketServer()` returns `undefined` until the HTTP server has
  started listening and also remains `undefined` when WebSockets are disabled
- production deployments must disable ops endpoints or protect them with
  `basic` or `jwt`

## Promotion Criteria for 1.0.0

`1.0.0-beta.1` must not be promoted to stable until:

- the example project is complete
- the example uses only the published package surface
- the example covers the supported feature set of `1.0.0`
- the documentation points to the example as the canonical starting point

See [releases.md](../releases.md) and [roadmap.md](../roadmap.md) for the
promotion path.
